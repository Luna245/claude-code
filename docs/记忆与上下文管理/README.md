## 核心要点（一分钟版）

如果你只读三句话，记住这三点：

1. **LLM 是无状态的**——每次请求把整个上下文重新发一遍，没有"记住"这回事。
2. **组件分三类**——始终在上下文里的（CLAUDE.md / MCP 定义 / Skill 元数据）、按需进入的（Skill 正文 / 工具调用结果）、永不进入的（Skill 脚本源码）。
3. **Prompt Cache 省的是钱和延迟，不是 token 数**——前缀字节不变就能命中，命中部分按 10% 价格计费、跳过 prefill。
