## 二、配置文件位置速查

| 文件 | 路径 | 作用范围 |
|------|------|---------|
| 全局 CLAUDE.md | `~/.claude/CLAUDE.md` | 所有项目 |
| 项目 CLAUDE.md | `<project>/CLAUDE.md` | 单个项目 |
| 跨工具约定 | `<project>/AGENTS.md` | 单个项目（支持其他工具） |

| 项目 Skill | `<project>/skills/<name>/SKILL.md` | 单个项目 |
| 全局 MCP | `~/.claude/mcp.json` | 所有项目 |
| 项目 MCP | `<project>/.mcp.json` | 单个项目 |
| Subagent | `<project>/.claude/agents/<name>.md` | 单个项目 |
| 用户设置 | `~/.claude/settings.json` | 所有项目 |
| 项目设置 | `<project>/.claude/settings.json` | 单个项目 |

## 四、settings.json 推荐模板

### 个人开发

```json
{
  "env": {},
  "permissions": {
    "allow": [
      "Bash(git diff:*)",
      "Bash(npm run lint:*)",
      "Bash(npm test:*)",
      "Read",
      "Edit"
    ],
    "deny": [
      "Bash(rm -rf:*)",
      "Bash(git push:*)"
    ]
  }
}
```

### 第三方平台（Bedrock/Vertex）兼容

```json
{
  "env": {
    "CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS": "1",
    "DISABLE_PROMPT_CACHING": "1"
  }
}
```

### 配 hook 自动 lint

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint --silent || true"
          }
        ]
      }
    ]
  }
}
```

## 八、常见问题排查

### Q1: Claude 不读我的 CLAUDE.md？

**检查清单**：
- 文件名拼写：必须是 `CLAUDE.md`（全大写）
- 位置：项目根目录或 `~/.claude/`
- 运行 `/status` 看 Claude Code 是否报告读到它
- 在 Claude Code 里输入 `/init` 重新初始化

### Q2: Skill 没被自动触发？

**可能原因**：
- description 写得太模糊（应该明确说"做什么场景下做什么"）
- 文件结构不对（必须有 `SKILL.md`，必须有 YAML front matter）
- 任务描述和 description 关联性弱

**调试方法**：直接说 "用 deploy-frontend skill 帮我部署"，强制触发。

### Q3: 上下文窗口爆了？

**优化路径**（成本由低到高）：
1. 删 CLAUDE.md 里冗余的内容
2. 把不常用的规则改成 Skill
3. 关闭不需要的 MCP server
4. 改用 Subagent 把大任务隔离出去
5. 用 `/compact` 主动压缩历史
6. 开新会话

### Q4: 缓存命中率很低？

**自查**：
- system prompt 里有动态内容（时间戳、随机 ID）？→ 挪走
- 经常切换 thinking 模式？→ 固定一个
- 来回开新会话？→ 保持长会话
- MCP server 重启频繁？→ 工具定义会变

### Q5: --resume 报 cache_control 错误？

第三方平台兼容问题。在 settings.json 里加：
```json
{
  "env": {
    "DISABLE_PROMPT_CACHING": "1"
  }
}
```

## 九、成本优化优先级

按性价比从高到低：

```
1️⃣  开启 Prompt Cache  ──→ 默认开，别关
2️⃣  控制 CLAUDE.md 长度 ─→ 别写成万字方针
3️⃣  精简 MCP server     ─→ 不用的就别启用，启动就吃几万 token
4️⃣  Skill 化偶发任务    ─→ 从 CLAUDE.md 挪到 Skill
5️⃣  长任务用 Subagent   ─→ 独立上下文不污染主会话
6️⃣  保持长会话          ─→ 别频繁开新窗口，复用 cache
```

## 十、性能/成本快速估算

### 一个"健康"的 Claude Code 会话

```
启动占用：
  System Prompt       ~3,000 token
  CLAUDE.md          ~1,000 token（建议上限）
  AGENTS.md          ~500 token
  3 个 MCP server    ~40,000 token
  10 个 Skill 元数据  ~1,000 token
  ─────────────────────────────
  小计               ~45,000 token（约 22% of 200k 窗口）

每轮新增：
  用户消息           ~100~500 token
  Claude 回复        ~200~1000 token
  可能的工具调用     ~500~5000 token
```

### 命中缓存 vs 未命中

```
对话第 50 轮，上下文累积 100k token：

未命中缓存（假设 Sonnet $3/M input）：
  100,000 × $3 / 1,000,000 = $0.30 / 请求

命中缓存（前缀 95k 缓存命中）：
  写入（第一次）：95,000 × $3.75/M = $0.356 
  读取（之后每次）：95,000 × $0.30/M = $0.0285
  + 动态部分：5,000 × $3/M = $0.015
  = 之后每次约 $0.044 / 请求

节省比例：约 85%
```

## 十二、写好 CLAUDE.md 的几条原则

1. **简明扼要**——能一句话说清楚的就一句话
2. **可执行**——"用 4 空格缩进" ✅ 而不是 "代码要好看" ❌
3. **避免冗余**——别和 system prompt 重复
4. **结构化**——用标题分组，方便 Claude 查找
5. **定期清理**——半年没用到的条目就删
6. **不要塞动态内容**——日期、用户名等会破坏缓存

### 反面教材

```markdown
# CLAUDE.md（不好）

请你做一个好的 AI 助手。
你应该写出高质量的代码。
注意要细心。
今天是 2026 年 5 月 30 日。   ← 破坏缓存
我叫张三，是个程序员。       ← 应该放别处
我们项目很重要。
...（继续 3000 行）
```

### 正面教材

```markdown
# CLAUDE.md（好）

## 项目栈
- TypeScript 5.x（strict 模式）
- React 19 + Tanstack Query
- Vitest 测试，禁用 Jest

## 风格
- 函数式优先，禁用 class
- 错误处理用 Result 类型
- 导入用 alias `@/`，禁用相对路径

## 约定
- 提交信息：Conventional Commits
- 分支：`feature/`、`fix/`、`chore/`
- PR：模板见 `.github/pull_request_template.md`
```
